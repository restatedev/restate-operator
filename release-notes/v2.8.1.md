# Restate Operator v2.8.1 Release Notes

## Highlights

- **Fix: the release build now works with the Rust 1.95.0 toolchain** — the release Docker base image is bumped from `ghcr.io/restatedev/dev-tools:1.16.1` to `1.17.0`, which bakes the Rust 1.95.0 toolchain and its musl standard library needed by the musl cross-compile.

## Bug Fixes

### Release build failed to cross-compile against musl

`rust-toolchain.toml` was bumped to 1.95.0 in v2.8.0, but the release Docker base
image `dev-tools:1.16.1` still baked an older toolchain and musl std. At build
time rustup re-downloaded 1.95.0 with `profile = minimal` and no `targets`, so the
musl target's `std` was missing and the cross-compile failed with
`error[E0463]` (can't find crate for `std`).

Bumping the base image to `dev-tools:1.17.0` — which bakes Rust 1.95.0 together
with the musl std, matching restate-server's toolchain pairing — restores the
release build.

This is a build/release-infrastructure patch only. There are no functional
operator changes versus v2.8.0.

## Upgrading

No CRD or functional changes in this release. Upgrade the operator via Helm:

```bash
helm upgrade restate-operator restatedev/restate-operator --version 2.8.1
```
