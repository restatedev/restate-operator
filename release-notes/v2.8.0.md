# Restate Operator v2.8.0 Release Notes

**v2.8.0 was never published — it was a broken release and is superseded by [v2.8.1](https://github.com/restatedev/restate-operator/blob/main/release-notes/v2.8.1.md).**

v2.8.0 attempted to gate CRD installation behind an `installCrds` value by *templating* the CRDs (moving them out of Helm's native `crds/` directory). That approach required manual CRD ownership adoption on upgrade (`invalid ownership metadata` errors) and, in its initial form, cascade-deleted custom resources on uninstall. It was abandoned before publication.

v2.8.1 ships the CRDs as an optional `restate-operator-crds` subchart using Helm's native `crds/` directory instead, still gated by `installCrds` (default `true`), which sidesteps both problems. See the [v2.8.1 release notes](https://github.com/restatedev/restate-operator/blob/main/release-notes/v2.8.1.md).
